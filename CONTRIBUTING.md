# Diretrizes de Contribuição

Obrigado por considerar contribuir com o Altair RPG! Este documento fornece as diretrizes para contribuir com o projeto.

## Como Contribuir

1. Faça um fork do projeto
2. Crie uma branch para sua feature (`git checkout -b feature/AmazingFeature`)
3. Faça commit das suas mudanças (`git commit -m 'Add some AmazingFeature'`)
4. Faça push para a branch (`git push origin feature/AmazingFeature`)
5. Abra um Pull Request

## Padrões de Código

### Geral
- Use TypeScript para todo código novo
- Mantenha a consistência com o estilo de código existente
- Adicione comentários para código complexo
- Escreva testes para novas funcionalidades

### Componentes React
- Use componentes funcionais com hooks
- Implemente tipagem adequada com TypeScript
- Siga o padrão de organização de arquivos existente
- Mantenha os componentes pequenos e focados

### Estilização
- Use TailwindCSS para estilização
- Siga o tema de cores definido no projeto
- Mantenha a consistência com o design system existente

## Commits

- Use mensagens de commit claras e descritivas
- Siga o padrão de commits convencionais:
  - `feat:` para novas features
  - `fix:` para correções de bugs
  - `docs:` para documentação
  - `style:` para mudanças de formatação
  - `refactor:` para refatorações
  - `test:` para adição/modificação de testes
  - `chore:` para tarefas de manutenção

## Pull Requests

- Descreva claramente as mudanças implementadas
- Referencie issues relacionadas
- Inclua screenshots para mudanças visuais
- Certifique-se que todos os testes passam
- Mantenha o PR focado em uma única funcionalidade

## Reportando Bugs

Ao reportar bugs, inclua:
- Descrição clara do problema
- Passos para reproduzir
- Comportamento esperado vs atual
- Screenshots (se aplicável)
- Ambiente (navegador, SO, etc)

## Sugerindo Melhorias

Para sugestões de melhorias:
- Descreva claramente a feature
- Explique por que ela seria útil
- Sugira uma implementação se possível
- Inclua mockups/wireframes se aplicável

## Dúvidas

Se tiver dúvidas sobre como contribuir, abra uma issue com a tag "question".
